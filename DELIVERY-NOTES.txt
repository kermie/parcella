DELIVERY NOTES -- Announcements module, foundation piece
======================================================================

New module for the announcements feature we discussed: authoring a
single piece of club news (Markdown body, optional image, optional
print-only shortened text) with a data model ready to track delivery
to three channels. This delivery is the FOUNDATION only -- authoring
and the data model. Nothing is sent anywhere yet: no blog draft, no
email, no PDF. Those are follow-up deliveries, each building on top of
what's here without changing its shape.

## What's included

- New tables: `announcements`, `announcement_deliveries`
  (migration 0029_announcements)
- New module flag `announcements`, OFF by default (same reasoning as
  public_signup_api -- it will hold outbound credentials and can
  eventually email every member, so it needs a deliberate opt-in)
- New admin-only pages under /announcements/ (list, create, edit) --
  restricted to admin/board, same as the Integrations page
- A Markdown editor (EasyMDE, live preview) for the body text
- Image upload per announcement (PNG/JPEG/WebP, up to 5 MB)
- All 7 language files updated (29 new keys each, verified matching)
- docs/module-announcements.md (full design) and a new section in
  docs/architecture-decisions.md
- tests/test_announcements.py (5 tests: module-flag gating, create/
  edit round-trip including Markdown->HTML rendering, validation,
  delete, and the admin/board-only permission boundary) -- all passing
  alongside the full existing suite (45/45)

## To apply

1. Unzip into your repo root (overwrites the files listed below,
   git status will show them as modified; adds the new files as
   untracked).
2. Run `pip install -r requirements.txt` (adds `markdown==3.7` -- the
   only new dependency).
3. Run the new migration: `alembic upgrade head`
   (0028_drop_signup_tables -> 0029_announcements). No data migration,
   no existing table touched -- purely additive.
4. Restart the app (Docker or otherwise) so the new router and
   templates are picked up.
5. The module stays invisible until you switch it on in
   Admin -> Settings ("Announcements"), same as Public Signup API.

No manual file deletions needed for this one -- everything here is new
or additive.
