DELIVERY NOTES -- Remove phone/email fields from the WordPress form
======================================================================

Confirmed: remarks were already flowing into the participant's Note
field on the session detail page (tested live: submitted a signup with
remarks, checked the API response -- it showed up exactly there,
alongside the submitted name). Nothing needed for that part.

Removed entirely, per your call that phone/email is redundant once a
signup resolves to a real Member record:
- Both input fields, gone from the form.
- The "Show phone number field" / "Show email address field" settings
  checkboxes, gone from Settings -> Parcella Connector (along with
  whatever was causing them not to save -- moot now, but my best guess
  was the hidden-field trick for unchecking wasn't the culprit; more
  likely something about how the settings API round-tripped the
  boolean value. Not worth chasing further since the feature's gone).
- The corresponding options (parcella_signup_show_phone /
  parcella_signup_show_email) are simply unused now; no cleanup step
  needed, WordPress just won't reference them anymore.
- Cleaned up the translation files (.po/.mo/.pot) to drop the six
  strings that were only used by these fields.

The form now collects: name (optional), parcel number (required),
work session checkboxes, and remarks (optional). The underlying
Parcella API still accepts optional phone/email in the signup payload
-- untouched, in case a different club's fork of this plugin still
wants them.

## To apply

Overwrite your existing plugin folder with this one (all 5 files
included). No database changes, no Parcella backend restart needed --
this is entirely inside the WordPress plugin.
