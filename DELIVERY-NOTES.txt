DELIVERY NOTES -- WordPress credentials moved to Integrations + plugin
consolidated into parcella-connector
======================================================================

Two changes this time, both from your feedback. No new migration, no
schema change.

## 1. WordPress blog credentials: Settings -> Integrations

Same three fields (site URL, username, Application Password), same
encrypted storage, same "blank password field = keep the existing
one" behavior -- just living on Admin -> Integrations now (a new card,
right below the public-signup API section) instead of Admin ->
Settings. The underlying database keys are unchanged, so if you'd
already saved credentials via Settings, they'll show up correctly
under Integrations without re-entering anything.

Admin -> Settings no longer has a WordPress card at all.

## 2. WordPress plugin: parcella-work-signup -> parcella-connector

Restructured into `integrations/wordpress/parcella-connector/`:
- `parcella-connector.php` -- shared settings screen (base URL + API
  token), a "Modules" table showing what's active
- `includes/modules/signup.php` -- the work-session signup shortcode,
  moved here unchanged in behavior
- Future integrations (tickets, applicants, calendar) become new files
  under `includes/modules/`, sharing the same settings screen -- no
  new credentials to configure per module

**If you'd already installed the old plugin** (sandbox or otherwise):
deactivate "Parcella Work Session Signup", install and activate
"Parcella Connector" instead. Your base URL and API token carry over
automatically -- I deliberately kept the underlying WordPress option
names identical. Any page using `[parcella_work_signup]` keeps working
unchanged; the shortcode tag didn't change either.

## Manual cleanup needed after unzipping

This one's on you to do manually, since I can't delete files outside
what I'm packaging: **delete the old
`integrations/wordpress/parcella-work-signup/` folder** from your repo
after unzipping this. It's fully superseded by parcella-connector; I
haven't included it in this ZIP, but it won't disappear from your
working copy on its own.

## Testing

10 tests updated/added: the two WordPress connection-test cases now
hit `/admin/integrations/wordpress/test` (was `/admin/settings/...`),
plus 3 new ones for the Integrations page itself -- saving works,
a blank Application Password field leaves the existing one alone, and
the saved password is never echoed back into the page HTML. 62/62
across the whole suite, all passing against real PostgreSQL.

The new PHP files were syntax-checked (`php -l`) and the German
translation catalog was compiled and round-trip verified (including
plural forms) using Babel, since msgfmt wasn't available in my
environment -- functionally identical output either way.

## To apply

1. Unzip into your repo root.
2. Delete the old `integrations/wordpress/parcella-work-signup/`
   folder (see above).
3. `docker compose up -d --build web` (no new dependency, but keeps
   things consistent).
4. No migration to run.
5. If WordPress credentials were already configured via Settings,
   nothing to do -- they'll appear under Integrations automatically.
   If not yet configured, fill them in there now.
6. On your WordPress site: swap the plugin as described above whenever
   convenient (not urgent -- the old plugin keeps working standalone
   until you do).
